## 2. 整体页面风格与设计规范 (Design Tokens)

为了给高校校友营造**真实、严肃、温馨且高质感**的交友氛围，小程序采用**现代极简 (Modern Minimal)** 风格，以温暖阳光的珊瑚色为基调，搭配细腻的间距与柔和的卡片投影。

### 2.1 颜色系统 (Color Palette)
开发必须严格使用以下预设的 CSS 自定义属性或 WXSS 常数：

```css
:root {
  /* 品牌核心色 */
  --color-primary: #FF7A59;         /* 珊瑚橙 (主色，用于主要按钮、高亮状态、认证图标) */
  --color-primary-light: #FFF0ED;   /* 浅蜜桃粉 (主要用于未读消息背景、轻量强调) */
  
  /* 辅助状态色 */
  --color-success: #10B981;         /* 森林绿 (用于认证成功标签) */
  --color-warning: #F59E0B;         /* 琥珀黄 (用于驳回警告、提示信息) */
  --color-error: #EF4444;           /* 警示红 (用于输入框报错、限制拦截状态) */
  
  /* 页面底色 */
  --color-background: #FFF9F6;      /* 蜜桃米/晨光白 (全局页面背景，带柔和微粉暖意) */
  --color-white: #FFFFFF;           /* 纯白 (用于卡片、弹窗、输入框的背景) */
  
  /* 文本色阶 */
  --color-text-primary: #1E293B;    /* 墨石蓝 (主标题、姓名、正文文本，高对比度) */
  --color-text-secondary: #64748B;  /* 石板灰 (次要辅助说明、表单标签、小眼睛数据) */
  --color-text-muted: #94A3B8;      /* 迷雾灰 (输入框占位符、小提示、已读轻辅助) */
  --color-text-disabled: #CBD5E1;   /* 冰川灰 (禁用状态文本) */
}
```

### 2.2 圆角体系 (Border Radius Scale)
严禁使用随意设置的 border-radius 值，统一遵循以下三个阶梯：
* **`border-radius: 8px`** (小圆角)：适用于小标签 (Tag)、输入框 (Input Box) 以及分类选择块等。
* **`border-radius: 12px`** (标准圆角)：适用于页面内主卡片 (Card) 容器、操作大按钮 (Button) 以及底部吸底栏的按钮。
* **`border-radius: 20px`** (大圆角)：适用于全局模态弹窗 (Modal Box) 以及具有沉浸式体验的大图画廊。

### 2.3 间距规范与排版网格 (Spacing & Grid)
排版网格遵循 **`4px` 步进基准**，以确保布局的节奏感与严谨性：
* **页面左右外边距**：统一为 **`16px`**，保证视觉在各种机型上都不贴边。
* **允许的间距集 (Spacing Set)**：`4px` / `8px` / `12px` / `16px` / `24px` / `32px` / `40px` / `48px`。所有 `margin`、`padding` 和 `gap` 均应取该集合中的值。
* **排版邻近性 (Proximity)**：相同逻辑组内的元素（如输入框与对应的小字提示）间距控制在 `4px` 或 `8px`；不同逻辑组之间的间距（如两个表单卡片）控制在 `16px` 或 `24px`。

### 2.4 字体字号规范 (Typography)
默认使用微信系统无衬线字体，字号梯度定义如下：
* **大标题 (Page Title)**: `18px` (Font Weight: Bold, 用于顶部 Header Title、登录欢迎、成功弹窗大字)。
* **中标题 (Card/Section Title)**: `15px` (Font Weight: Medium/Bold, 用于卡片内人名、活动标题、表单区块大标题)。
* **正文 (Body Text)**: `13px` (Font Weight: Normal, 用于自我介绍文本、破冰回答文本、说明文、常规输入内容)。
* **辅助说明/标签 (Caption/Tag)**: `11px` ~ `12px` (Font Weight: Normal/Bold, 用于输入框下方提示、已认证小字标签、小眼睛浏览数据)。
* **微型元数据 (Tiny)**: `10px` (Font Weight: Bold, 用于步骤指示条下方数字、未读红点数字)。

### 2.5 阴影体系 (Shadows)
为避免沉闷和脏感，页面卡片不得使用深黑色或大范围重阴影。统一采用极细微带有珊瑚色偏移的软投影：
* **卡片常规投影**：`box-shadow: 0 4px 12px rgba(255, 127, 80, 0.06);`

## 3. 组件级设计规范 (Component Specs)

### 3.1 输入框与文本域 (Inputs & TextAreas)
所有输入组件必须具有明确的状态感知，高亮指示当前聚焦区：
* **常态 (Inactive)**：白底，带有细边框 `1px solid #E2E8F0`，圆角 `8px`。占位符字体为 `13px` 迷雾灰 (`--color-text-muted`)。
* **聚焦态 (Active/Focus)**：激活细边框 `1.5px solid var(--color-primary)`，其余不变，无其他多余阴影，防止视觉过载。
* **异常态 (Error)**：触发校验失败时，边框变红 `1.5px solid var(--color-error)`，并于输入框下方显示红色警示小字。

### 3.2 按钮体系 (Buttons)
为保障视觉克制力，主操作渐变至多每页出现一个：
1. **主行动键 (Primary CTA Button)**：
   - 样式：珊瑚橙微渐变背景 (`background: linear-gradient(135deg, #FF9E84, #FF7A59)`)，高度 `44px` 或 `48px`，圆角 `12px`。
   - 文本：`14px`/`15px` 粗体白字。
   - 交互：支持微信反馈震动。禁用状态时背景为灰色 `#E2E8F0`，文本为 `#94A3B8`，且无法触发点击。
2. **次行动键 (Secondary Button)**：
   - 样式：纯白底色，细边框 `1px solid var(--color-primary)`，高度相同。
   - 文本：`14px`/`15px` 珊瑚橙色字。

### 3.3 图标系统 (Iconography)
* **硬性规则**：**UI 中禁止直接使用 Emoji 表情作为界面图标或装饰**（如 PRD 提及的 📅 统一替换为日历矢量图标，🔒 替换为小锁矢量图标）。
* **图标库**：全系统必须引入同一套 Lucide 矢量图标，保持风格完全一致（粗细均为 `2px`，无填充）。
* **尺寸分类**：
  - 顶部返回键、底栏 Tab 图标：尺寸为 `24px * 24px`。
  - 卡片内部小眼睛（浏览）、同频（点赞）图标、表单内小缩进图标：尺寸为 `16px * 16px`。

### 3.4 模态弹窗规范 (Modal / Dialog Box)
* **版面布局**：宽固定为 `330px`，高度自适应，圆角 `20px`。
* **黑色遮罩**：背景加设磨砂半透明遮罩（`background: rgba(0, 0, 0, 0.5); backdrop-filter: blur(4px);`）。
* **结构规范**：头部居中标题（加粗 primary 颜色），中间垂直排列辅助文案或输入表单，底部横向排布或单行大按钮，操作界限必须清晰分明。

## 4. 统一错误提示与异常反馈机制

开发团队在处理用户操作异常或逻辑拦截时，必须遵循以下四种场景的统一反馈规范：

### 4.1 场景一：表单输入实时校验与报错
* **适用情况**：在认证建档页中，如身份证号格式错误、手机号不足位、心动问题回答少于 20 字等。
* **反馈规范**：
  - 对应的输入框或文本域瞬间标记为**异常态（红边框）**。
  - 在输入组件正下方 `4px` 处，垂直显现 `11px` 红字 `* [具体错误描述，指出如何修正]`（例如：`* 破冰回答需至少包含 20 个字，当前已输入 12 字`）。
  - 页面主提交按钮自动更新为禁用置灰状态。

### 4.2 场景二：全局接口或网络请求报错
* **适用情况**：云函数调用失败、网络异常超时、Redis 实名核验日限额报错等。
* **反馈规范**：
  - **不影响当前页面时**：使用半透明黑底 Toast 悬浮通知（背景：`rgba(0,0,0,0.8)`，圆角 `8px`），在屏幕中央展示 `3秒` 自动消隐，文字力求短小明确（如：`网络连接超时，请稍后再试`）。
  - **影响页面提交时**：在屏幕正顶部 Header 栏下方浮出淡红 Alert 提示条（背景：`#FEE2E2`，文字为 `--color-error`），提示语具有引导性（如：`身份证比对通道拥堵，系统将自动转入人工复核通道`）。

### 4.3 场景三：人工审核不通过/被驳回异常提示
* **适用情况**：用户再次打开小程序，被管理员判定学历/工作证明不合规而被驳回。
* **反馈规范**：
  - 在个人中心 Tab 屏（Me Screen）的最顶部，动态渲染一块**黄色背景卡片**（背景：`#FFFBEB`，边框：`1px solid #FEF3C7`，圆角 `12px`）。
  - 卡片内以琥珀黄字展示：“⚠️ 您的[学历认证]申请被驳回。原因：[驳回原因文本]”，右侧附带一个点击区：“去修改”。
  - 用户点击“去修改”时，直接高亮跳转回认证建档页对应的步骤，且该步骤页面将红字高亮并反填之前的输入。

### 4.4 场景四：未认证用户的权限拦截
* **适用情况**：跳过认证的试用用户试图点击瀑布流详情、点赞、发起心动或报名活动。
* **反馈规范**：
  - 系统触发拦截手势。
  - 弹出标准的 Dialog 拦截弹窗：
    * 标题：`“开启完整校缘之旅”`
    * 描述：`“为了保障校友相亲的真实度与严肃性，查看详情、点赞及心动匹配需要先完成实名与学校认证。只需 2 分钟，即可结识南京大学/江南大学的优秀同学。”`
    * 底部按钮：左侧 `“暂不认证”`（灰色），右侧主按钮 `“立即去认证”`（珊瑚橙高亮）。点击右侧主按钮跳转至认证引导页 `/pages/verification/index`。
